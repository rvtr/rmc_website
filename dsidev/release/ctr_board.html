Thanks to Emily, we have a new CTR BOARD! This tool would have been used in factories to test 3DS motherboard, microphones, speakers, IR communications, and the gyroscope. BOARD would be run from a development Nintendo DSi, connected to the 3DS through a special testing rig. Photos of the completed setup are shown below. The specific version of BOARD in this release was used exclusively to test CTR (old 3DS) models. This is a development signed ROM.

There are two main modes of this software: BOARD, and FINAL. BOARD tests a loose motherboard at the start of the factory process. This checks for basic things like power management. FINAL, like its name would suggest, is used when the 3DS has been completed. This mode tests the peripherals like speakers, microphones, and IR.

Unlike most TWL EVAs, you cannot access the menu by holding START on boot. The BOARD software automatically tries communicating to an XPort device. Since we do not have the factory hardware needed for communications, the software will be stuck trying to load until it eventually times out. This can take a few minutes, and the main menu will not be available until loading completes or fails. You may turn off framerate limiting in melonDS to speed up the process.
