PokeWalker Factory Tests

Today we're releasing two versions of the PHC PRODUCT TEST, a factory program for the PokeWalker (PHC) hardware. The tests are run from a Nintendo DS, with the PokeWalker connecting over infrared. Special dev cartridges are used, modified with an added IR transmitter on the save module, and a translucent black cover to allow IR light to pass through.

PHC PRODUCT TEST comes in two modes: WRITE and VERIFY. Both were compiled on 2009/05/18.

  • The WRITE mode will write to the PokeWalker's EEPROM with UNIQUE ID DATA (this sets up RTC), then LCD INIT DATA.
  • The VERIFY mode will just verify these two datas.
In order for the PokeWalker to connect to PHC PRODUCT TEST, you will need to reset the pokewalker with the test software open. When PHC PRODUCT TEST gets run, it sets the PokeWalker into a special mode called interactionHandlerDebugMode22 which allows communication with more advanced factory hardware through SPI. From here the PokeWalker will run EEPROM, RTC, and accelerometer tests. (source)
